# API Scraper (http-stream)

## Обновленный алгоритм
В этой ветке алгоритм парсинга решил улучшить - парсить в одном Message на 1 пост, а несколько. Заметил интересную
деталь - если одним Message обрабатывать 10 постов, и если в супервизоре запустить >25 воркеров - API сервер падает.

## Результаты
Сейчас по скорости парсинга он немножко уступает алгоритму из мастер ветки - пока смог выжать только ~60000 за 5 часов, 
возможно стоит поиграться с параметрами - количеством воркеров супервизора и размером батча на один Message
